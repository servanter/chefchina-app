# 测试指南 - 返回按钮功能验证

## 🎯 测试目标

验证 3 个营养档案页面的返回按钮功能和视觉效果是否符合设计要求。

---

## 📋 测试环境

### 必需条件
- [ ] React Native 开发环境已配置
- [ ] Node.js 和 npm 已安装
- [ ] Expo CLI 已安装
- [ ] iOS 模拟器或 Android 模拟器已启动 (或真机连接)

### 启动应用
```bash
cd chefchina-app
npm install
npm start
```

选择平台:
- 按 `i` - iOS 模拟器
- 按 `a` - Android 模拟器
- 扫码 - 真机测试

## 🎨 视觉验证清单

### 按钮样式
- [ ] 宽度: 36px

- [ ] 圆角: 18px (完美圆形)
- [ ] 背景色: #F5F2EE (浅米色)
- [ ] 图标: arrow-back (向左箭头)
- [ ] 图标大小: 20px
- [ ] 图标颜色: #333 (深灰色)

### 标题样式
- [ ] 字体大小: 20px
- [ ] 字重: bold
- [ ] 颜色: #333
- [ ] 对齐: 居中
- [ ] 位置: 在返回按钮和右侧占位之间

### 布局
- [ ] 返回按钮在左侧

### 间距
- [ ] 水平内边距: 16px (左右)

## 🔄 导航流程测试

### 测试场景 A: 顺序导航
```
主页 → 营养档案设置 → [返回] → 主页 ✓
主页 → 每日记录 → [返回] → 主页 ✓
主页 → AI 周报 → [返回] → 主页 ✓
```

### 测试场景 B: 快捷入口导航
```
主页 → 营养档案设置 → 每日记录 → [返回] → 营养档案设置 ✓
营养档案设置 → AI 周报 → [返回] → 营养档案设置 ✓
```

### 测试场景 C: 深度导航
```
主页 → 营养档案设置 → 每日记录 → [记录按钮] → 
记录模态框 → [关闭] → 每日记录 → [返回] → 
营养档案设置 → [返回] → 主页 ✓
```

**验证点**:
- [ ] 返回按钮总是返回到正确的上一级页面
- [ ] 导航栈管理正确,没有页面堆叠问题
- [ ] 返回速度流畅,无卡顿
